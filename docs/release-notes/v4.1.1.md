# v4.1.1

## 概要

這個版本聚焦在 sitemap 與專案展示資料的整理，主要優化搜尋引擎可讀性、穩定 sitemap metadata，並同步更新個人網站專案卡片與版本號。整體目標是讓 SEO 設定更準確、內容展示更一致。

## 亮點更新

### 1. sitemap metadata 最佳化

- 調整 `sitemap.xml` 的 `lastModified` 產生邏輯，優先使用文章 `updateDate`，沒有時再回退到 `date`。
- 補上穩定的站點啟用時間常數，避免 sitemap 在缺少內容時使用浮動時間。
- 將 `/blog` canonical 列表頁納入 sitemap，讓搜尋引擎更容易理解主要內容入口。

### 2. 固定頁 sitemap 行為更穩定

- 將 `/about` 頁面的 `lastModified` 改成穩定值，不再每次建置都看起來像剛更新。
- 統一首頁、分類頁與文章頁的 sitemap 時間策略，讓整體訊號更一致。

### 3. 專案卡片資料更新

- 更新個人網站專案卡片的 `id`、圖片、連結與顯示名稱。
- 專案儲存庫連結同步改為 `garylin.dev` 新位置。
- 對應更新專案圖片資產名稱，讓展示內容與資料來源一致。

## 內部重構與維護性調整

- `robots.txt` 持續維持指向 `${DOMAIN}/sitemap.xml` 的 sitemap discovery 設定。
- 同步更新 Velite 產物，讓編譯輸出與內容資料保持一致。
- 專案版本號更新為 `4.1.1`。

## 可能的破壞性變更

這個版本以 sitemap 與專案資料調整為主，沒有預期的破壞性變更。

- 搜尋引擎之後重新抓取 sitemap 時，可能會看到更新後的 `lastModified` 訊號。
- 個人網站專案卡片的圖片路徑與 repo 連結已更新為新名稱與新位置。
