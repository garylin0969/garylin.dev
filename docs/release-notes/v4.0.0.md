# v4.0.0

## 概要

升級到 `Next.js 16.2.0`、導入 agent / skill 相關規則與資產、首頁內容改由 Velite 管理，以及整體部落格與導覽體驗的重構。整體目標是讓網站更容易維護、內容來源更一致，並改善使用者在桌機與手機上的瀏覽體驗。

## 亮點更新

### 1. 升級到新版 Next.js

- 專案已升級到 `Next.js 16.2.0`。
- 同步調整部分頁面結構與實作方式，以符合新版 App Router、建置流程與框架行為。

### 2. 導入 agent / skill 開發規範

- 新增 `.agents/` 相關規則與 skills 資產。
- 補上 `AGENTS.md`、`CLAUDE.md`、`skills-lock.json` 等開發輔助設定。
- 讓後續在本專案中的 AI 協作、規則套用與開發流程更一致。

### 3. 首頁內容改由 Velite 管理

- 首頁內容已改為由 Velite 單一內容檔案提供。
- 讓首頁與文章內容採用一致的內容管理方式。
- 後續調整首頁文案時，只需要更新內容檔即可。

### 4. 首頁與 About 頁重新整理

- 新增並重構首頁內容展示元件。
- 調整 About 頁的區塊樣式與內容排列。
- 讓整體視覺與資訊層級更一致。

### 5. 部落格閱讀體驗改善

- 文章標題與目錄改用穩定的錨點 ID。
- 改善文章日期語意標記與導覽可及性。

### 6. 行動裝置導覽優化

- 手機版 Navigation 改為更穩定的 client-side sheet 控制。
- 在手機版點選導覽項目後，選單會自動關閉。

## 內部重構與維護性調整

- 移除未使用的 blog stats 與相關展示元件。
- 清理未使用常數與不再需要的元件 barrel export。
- 調整文章排序邏輯，避免直接修改共享資料。
- 將 heading slug 規則抽成共用工具，減少重複邏輯。

## 可能的破壞性變更

由於這是 `4.0.0` 的主要版本更新，以下部分與舊版相比有明顯變化：

- 首頁資料來源已切換為 Velite。
- 專案已升級到 `Next.js 16.2.0`，部分內部實作與舊版不相容。
- 新增 agent / skill 規則與資產，開發流程與專案約束比舊版更完整。
- 文章 heading 結構新增 `id` 欄位。
- 部分未使用元件與常數已移除。
- 專案儲存庫參考已更新為 `garylin.dev`。
